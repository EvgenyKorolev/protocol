Обозначение переменных для получения данных родительских объектов:

Переменные с данными о клиенте:

  1. Переменные клиента
    1. _kl_name - имя клиента
    2. _kl_full_name - полное наименование клиента
    3. _kl_email - email клиента
    4. _kl_det_inn - ИНН клиента
    5. _kl_det_dirpos - название должности руководителя
    6. _kl_det_dirname - ФИО руководителя
  2. Телефоны клиента:
    1. _kl_det_tel - телефон если есть
    2. _kl_det_tel1 - второй телефон если есть
    3. _kl_det_tel2 - третий телефон если есть
    4. _kl_det_tel3 - четвертый телефон если есть
    5. _kl_det_tel4 - пятый телефон если есть
  3. Факсы клиента:
    1. _kl_det_fax - факс если есть
    2. _kl_det_fax1 - второй факс если есть
    3. _kl_det_fax2 - третий факс если есть
    4. _kl_det_fax3 - четвертый факс если есть
    5. _kl_det_fax4 - пятый факс если есть
  4. Адрес почтовый:
    1. _kl_padr_f1 - адрес целиком
    2. _kl_padr_f2 - адрес целиком в другом формате
    3. _kl_padr_f3 - адрес целиком в другом формате
    4. _kl_padr_cont - страна нахождения клиента
    5. _kl_padr_statclass - вид региона клиента (республика, область)
    6. _kl_padr_state - название региона клиента
    7. _kl_padr_stdistr - название района региона клиента
    8. _kl_padr_citycl - вид населённого пункта клиента
    9. _kl_padr_city - назвние населённого пункта клиента
    10. _kl_padr_distr - район расположение (города) клиента
    11. _kl_padr_strcl - вид улицы клиента (проспект, тупик)
    12. _kl_padr_str - название улицы клиента
    13. _kl_padr_bnum - номер строения клиента
    14. _kl_padr_blit - литер строения клиента
    15. _kl_padr_ofnum - номер помещения клиента
    16. _kl_padr_oflit - литер помещения клиента
    17. _kl_padr_postind - почтовый индекс клиента
  5. Адрес юридический:
    1. _kl_uadr_f1 - адрес целиком
    2. _kl_uadr_f2 - адрес целиком в другом формате
    3. _kl_uadr_f3 - адрес целиком в другом формате
    4. _kl_uadr_cont - страна нахождения клиента
    5. _kl_uadr_statclass - вид региона клиента (республика, область)
    6. _kl_uadr_state - название региона клиента
    7. _kl_uadr_stdistr - название района региона клиента
    8. _kl_uadr_citycl - вид населённого пункта клиента
    9. _kl_uadr_city - назвние населённого пункта клиента
    10. _kl_uadr_distr - район расположение (города) клиента
    11. _kl_uadr_strcl - вид улицы клиента (проспект, тупик)
    12. _kl_uadr_str - название улицы клиента
    13. _kl_uadr_bnum - номер строения клиента
    14. _kl_uadr_blit - литер строения клиента
    15. _kl_uadr_ofnum - номер помещения клиента
    16. _kl_uadr_oflit - литер помещения клиента
    17. _kl_uadr_postind - почтовый индекс клиента
  6. Адрес фактический:
    1. _kl_fadr_f1 - адрес целиком
    2. _kl_fadr_f2 - адрес целиком в другом формате
    3. _kl_fadr_f3 - адрес целиком в другом формате
    4. _kl_fadr_cont - страна нахождения клиента
    5. _kl_fadr_statclass - вид региона клиента (республика, область)
    6. _kl_fadr_state - название региона клиента
    7. _kl_fadr_stdistr - название района региона клиента
    8. _kl_fadr_citycl - вид населённого пункта клиента
    9. _kl_fadr_city - назвние населённого пункта клиента
    10. _kl_fadr_distr - район расположение (города) клиента
    11. _kl_fadr_strcl - вид улицы клиента (проспект, тупик)
    12. _kl_fadr_str - название улицы клиента
    13. _kl_fadr_bnum - номер строения клиента
    14. _kl_fadr_blit - литер строения клиента
    15. _kl_fadr_ofnum - номер помещения клиента
    16. _kl_fadr_oflit - литер помещения клиента
    17. _kl_fadr_postind - почтовый индекс клиента

Переменные с данными о заявке:

  1. _ord_num - номер завки
  2. _ord_uniq - название завки
  3. _ord_date - дата завки
  4. _ord_date_1 - дата завки в формате dd.MM.yyyy
  5. _ord_date_2 - дата завки в формате dd.MM.yy
  6. _ord_date_3 - дата завки в формате dd.MMM.yyyy
  7. _ord_type - возвращает "ord"

Переменные с данными об объекте:

  1. Переменные объекта (ЦП или КТП):
    1. _obj_name - имя объекта (ЦП или КТП)
    2. _obj_unom - номинальное напряжение
    3. _obj_sh - параметр СШ
    4. _obj_app - таблица приборов (можно указать стиль в формате CSS в параметре style) объекта
    5. _obj_app1 - таблица приборов (можно указать стиль в формате CSS в параметре style) объекта, другой формат времени
    6. _obj_app2 - таблица приборов (можно указать стиль в формате CSS в параметре style) объекта, другой формат времени
    7. _obj_app3 - таблица приборов (можно указать стиль в формате CSS в параметре style) объекта, другой формат времени
    8. _obj_app4 - таблица приборов (можно указать стиль в формате CSS в параметре style) объекта, другой формат времени
  2. Адрес объекта:
    1. _obj_padr_cont - страна нахождения объекта
    2. _obj_padr_statclass - вид региона объекта (республика, область)
    3. _obj_padr_state - название региона объекта
    4. _obj_padr_stdistr - название района региона объекта
    5. _obj_padr_citycl - вид населённого пункта объекта
    6. _obj_padr_city - назвние населённого пункта объекта
    7. _obj_padr_distr - район расположение (города) объекта
    8. _obj_padr_strcl - вид улицы объекта (проспект, тупик)
    9. _obj_padr_str - название улицы объекта
    10. _obj_padr_bnum - номер строения объекта
    11. _obj_padr_blit - литер строения объекта
    12. _obj_padr_ofnum - номер помещения объекта
    13. _obj_padr_oflit - литер помещения объекта
    14. _obj_padr_postind - почтовый индекс объекта
  3. Данные о трансформаторе
    1. _trans_type - тип трансформатора
    2. _trans_nom - заводской номер трансформатора
    3. _trans_class - класс точности трансформатора
    4. _trans_npmn - номинальная полная мощьность нагрузки трансформатора Sh для указанного класса точности трансформатора
    5. _trans_cco - Схема соединения обмоток (усл. обозначение)
    6. _trans_nnov1 - номинальное напряжение основной вторичной обмотки U2 трансформатора
    7. _trans_nnov2 - номинальное напряжение основной вторичной обмотки U1 трансформатора
    8. _trans_faz - количество фаз трансформатора
