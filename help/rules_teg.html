Тэги для подстановки данных в протокол

Тег записывется в виде: <имя_тэга type = "тип_тэга", message = "поясняющее_сообщение", vname = "имя_переменной"> где:

имя_тэга:

  1. _parent_var - тэг для подстановки значений из родительских объектов (клиент, заявка, ЦП, КТП) или лаборатории
    1. Типы (type = ):
      1. klient - данные клиента
      2. ord - даные заявки
      3. ktp - данные КТП
      4. cp - данные ЦП
      5. obj - данные объекта (ЦП или КТП) родителя
      6. lab - данные лаборатории
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - имя переменной значением которой будет замене тэг
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных
  2. _prot_con - Задаёт переменную значение котрых вводится в конструкторе протоколов при формировании
    1. Тип (type = ) - тип переменной
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - имя переменной
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных
  3. _prot_type - тэг для подстановки значений зависящих от типа протокола
    1. Тип (type = ) - совпадает с именем переменной для протокола.
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - имя переменной значением которой будет замене тэг
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных
  4. _const_type - тэг для подстановки значений зависящих от установленных наборов констант
    1. Тип (type = ) - совпадает с именем переменной для набора констант.
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - имя переменной значением которой будет замене тэг
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных
  5. _set_adapt_var - Задаёт переменную которую надо поместить в список переменных для скриптов
    1. Тип (type = ) - имя переменной которое будет доступно скриптам
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - значение переменной которое будет доступно скриптам
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных
  6. _ask_obj - Задаёт переменную значение котрой программа запросит у пользователя
    1. Тип (type = ) - Имя типа набора, в котором поиск значения ведётся в первую очередь
    2. Сообщение (message = ): - текстовое сообщений выводящееся при запросе значения переменной у пользователя
    3. Имя_переменной (vname = ) - имя переменной которая будет запрошена
    4. Стиль (style = ) - Параметры интерпритация которых зависит от контекста и влияет на отображение данных

В теге _ask_obj атрибут type определяет класс набороа констант. Если выбрать 2 набора констант одного класса, то будет учтён только один из них. Логика работа следующая: для подходящих близких по смыслу групп данных прописываются тэги с разным классом. При формировании протокола можно выбрать несколько наборов констант, по одному для каждого класса.
